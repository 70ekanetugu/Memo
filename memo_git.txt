～Git個人メモ～
[ワークフロー]
１．リポジトリの作成　Git & GitHub
２．ファイルの作成／変更
３．ステージエリアへ"add"
４．ローカルリポジトリへ"commit"
５．共有リポジトリへ"push"

[基本コマンド]
・git --help　->　ヘルプの表示

・git add　"ファイル名" -> ファイルをステージエリアへ追加する。
　　　　　　　　　　　　"ファイル名"を"."にすると、ディレクトリ内のファイルが全て追加される。

・git rm	　　　　　　->　ファイルを削除する。削除後、commitも必要。

・git commit ->　ステージからローカルリポジトリへファイルの変更(差分)をタイムスタンプと一緒に記録する。

・git status　->　現在のファイル変更状況を表示する。
　　　　　　　　　ステージに追加されていない変更ファイルがある場合、”赤文字”
　　　　　　　　　ステージに追加されているがコミットされていないファイルがある場合は、"緑文字"

・git log ->　これまでのログを表示する。
