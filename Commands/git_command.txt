～Git個人メモ～
[ワークフロー]
１．ローカルリポジトリの作成　    "git init "
２．リモートリポジトリの作成　     GitHub上で行う。
３．ファイルの作成／変更／削除
４．ステージエリアへ　            "git add ファイル名(.全ファイル)"
５．ローカルリポジトリへコミット　"git commit"
６．リモートリポジトリの登録　    "git remote add origin ﾘﾓｰﾄﾘﾎﾟｼﾞﾄﾘのURL"
７．共有リポジトリへ              "git push origin master"
     --->username & passwordの入力


[基本コマンド]
・git --help　
	->　ヘルプの表示

・git init
	->ローカルリポジトリの作成コマンド。
	　作りたいディレクトリに移動しておく事！	　

・git add　"ファイル名" 
	-> ファイルをステージエリアへ追加するコマンド。
　　　　　"ファイル名"を"."にすると、ローカルリポジトリ内ディレクトリ内のファイルが全て追加される。

・git rm
　　->　ファイルを削除する。削除後、commitも必要。

・git commit 
	->ステージからローカルリポジトリへファイルの変更(差分)
	　をタイムスタンプと一緒に記録する。

・git status　
	->現在のファイル変更状況を表示する。
　　　　　ステージに追加されていない変更ファイルがある場合、”赤文字”
　　　　　ステージに追加されているがコミットされていないファイルがある場合は、"緑文字"

・git log 
	->これまでのログを表示する。

・git diff
	->変更差分の表示

・git remote add "origin" "リモートリポジトリURL"
	->リモートリポジトリを登録する。(エイリアス)
	　エイリアス名は何でもいいが、originにするのが一般的。
　　　　　URLはGitHub上で作成したリポジトリのURL。

・git push　origin "master / branch"
	->リモートリポジトリにファイルを追加する。
　　　　　追加するファイルは現在いるローカルリポジトリのもの。
